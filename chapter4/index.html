    Chapter 4: Standard Image Preprocessing¶

    ‚ÄúA well-preprocessed image is half the training. The other half is just optimization.‚Äù

    Why This Chapter Matters¶

    Imagine handing a blurry, off-centered photo to a human and asking, ‚ÄúWhat‚Äôs in this?‚Äù You‚Äôd expect a confused answer‚Äîand that‚Äôs exactly how neural networks feel when they receive poorly scaled, misaligned, or inconsistent input images.

    Preprocessing is the discipline of preparing your data in a way the model can truly understand. It‚Äôs not just resizing or flipping‚Äîit‚Äôs about standardizing, cleaning, and augmenting images so your CNN can extract meaningful patterns.

    Done right, preprocessing:

    • Boosts model accuracy

    • Reduces overfitting

    • Makes inference stable

    • Speeds up convergence

    Done wrong? You‚Äôll spend weeks tuning your architecture for nothing.

    Conceptual Breakdown¶

    üîπ What Is Image Preprocessing?

    Image preprocessing refers to the transformations applied to raw image data before it‚Äôs fed into the model. It‚Äôs the very first thing your pipeline does, and its job is to:

    • Resize or crop the image to fit the model‚Äôs input size

    • Convert the pixel values to float and normalize them

    • Augment images with randomness during training to improve generalization

    üîπ Common Preprocessing Operations

    Step Purpose
    Resize Match input shape expected by CNN (e.g., 224√ó224)
    Crop Focus on central content or apply randomness
    Normalize Scale values for model stability and consistency
    Augment Random changes (flip, rotate, jitter) to generalize better

    üîπ Mean-Std Normalization vs 0‚Äì1 Scaling

    You‚Äôll often see two types of normalization:

    1. 0‚Äì1 scaling: divide pixel values by 255

      • Simpler, used for custom models

    2. Mean-std normalization: subtract dataset mean, divide by std

      • Used with pretrained models (e.g., ResNet, MobileNet)
    Format Example
    0‚Äì1 Scaling img = img / 255.0
    Mean-std Norm img = (img - mean) / std
    ImageNet Mean [0.485, 0.456, 0.406]
    ImageNet Std [0.229, 0.224, 0.225]

    üîπ Effects of Preprocessing on Training

    Preprocessing Problem Symptoms
    No normalization Model fails to converge
    Wrong mean/std Bad predictions, poor transfer
    Mismatched resize Shape errors at input layers
    Augmenting test data Erratic evaluation accuracy

    Note: Always apply augmentation to training data only, never to validation/test.

    üîπ PIL vs OpenCV vs tf.image

    Library Style Format Used
    PIL Pythonic RGB (default)
    OpenCV Fast, C++ BGR (must convert!)
    tf.image Tensor-native TensorFlow Tensors

    If you‚Äôre mixing OpenCV with TensorFlow or PyTorch, be careful‚Äîcolor channels will be flipped unless you convert BGR ‚Üí RGB.

    üîπ Preprocessing Matching: Train vs Inference Your model learns with a certain set of expectations (shape, scale, mean/std). If your inference pipeline doesn‚Äôt match your training pipeline, your model will:

    Output low-confidence predictions

    Misclassify even familiar data

    üìå Golden Rule: Always reuse your training preprocessing (minus augmentation) for inference.

    PyTorch Implementation¶

    üî∏ Training Preprocessing 

    from torchvision import transforms

train_transform = transforms.Compose([
    transforms.Resize((256, 256)),
    transforms.RandomCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),  # scales to [0, 1]
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])

    üî∏ Validation / Inference Preprocessing 

    val_transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])

    TensorFlow Implementation¶

    üî∏ Training Preprocessing 

    import tensorflow as tf

def preprocess_train(img):
    img = tf.image.resize(img, [256, 256])
    img = tf.image.random_crop(img, [224, 224, 3])
    img = tf.image.random_flip_left_right(img)
    img = tf.cast(img, tf.float32) / 255.0
    mean = tf.constant([0.485, 0.456, 0.406])
    std = tf.constant([0.229, 0.224, 0.225])
    img = (img - mean) / std
    return img

    üî∏ Inference Preprocessing 

    def preprocess_eval(img):
    img = tf.image.resize(img, [224, 224])
    img = tf.cast(img, tf.float32) / 255.0
    mean = tf.constant([0.485, 0.456, 0.406])
    std = tf.constant([0.229, 0.224, 0.225])
    img = (img - mean) / std
    return img

    Framework Comparison Table¶

    Task PyTorch TensorFlow
    Resize transforms.Resize() tf.image.resize()
    Crop RandomCrop, CenterCrop tf.image.random_crop()
    Flip RandomHorizontalFlip() tf.image.random_flip_left_right()
    Normalize transforms.Normalize(mean, std) Manual: (img - mean) / std
    Convert to tensor transforms.ToTensor() tf.cast(img, tf.float32) / 255.0
    Augment only in training Manual via Compose Wrap in @tf.function or dataset map

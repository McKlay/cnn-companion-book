    Chapter 6: Image Datasets ‚Äì Getting Data Into the Network¶

    ‚ÄúIt‚Äôs not just about images‚Äîit‚Äôs about structure, batching, and flow. A CNN needs data, not chaos.‚Äù

    Why This Chapter Matters¶

    You‚Äôve learned how to preprocess a single image, but deep learning thrives on batches of thousands of images‚Äîefficiently streamed, augmented, shuffled, and labeled.

    If your dataset isn‚Äôt:

    • Properly structured

    • Mapped to class labels

    • Efficiently loaded and shuffled

    ‚Ä¶ then even the best model won't help you. You‚Äôll face:

    • GPU idle time

    • Overfitting from data leaks

    • Mislabeling issues

    This chapter shows how to transform folders of images into usable model inputs, with complete control over batching, shuffling, label handling, and augmentation.

    Conceptual Breakdown¶

    üîπ The Canonical Folder Structure

    Most frameworks expect images arranged like this: 

    dataset/
‚îú‚îÄ‚îÄ train/
‚îÇ   ‚îú‚îÄ‚îÄ class1/
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ img1.jpg
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ img2.jpg
‚îÇ   ‚îú‚îÄ‚îÄ class2/
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ img3.jpg
‚îÇ   ‚îÇ   ‚îú‚îÄ‚îÄ img4.jpg
‚îú‚îÄ‚îÄ val/
‚îÇ   ‚îú‚îÄ‚îÄ class1/
‚îÇ   ‚îú‚îÄ‚îÄ class2/
    Each folder name becomes the class label, and all images inside are automatically mapped.

    This structure is:

    • Easy to understand

    • Supported natively in both PyTorch and TensorFlow

    • Flexible for augmentation and splitting

    üîπ Dataset Terms You Must Know

    Term Definition
    Dataset Collection of images and labels (can be lazy-loaded)
    Dataloader Handles batching, shuffling, and parallel loading
    Transform Applied per sample to augment/normalize
    Batch A group of N samples fed to the model at once
    Epoch One full pass through the dataset

    üîπ Why Shuffling Matters

    Shuffling prevents:

    • Learning label order bias (e.g., all cats, then all dogs)

    • Memorizing batch patterns

    • Overfitting on dataset structure

    üìå Always shuffle during training, not validation.

    üîπ Visualizing a Batch

    Understanding shapes is critical.

    Description PyTorch Shape TensorFlow Shape
    One RGB image [3, H, W] [H, W, 3]
    One batch (N images) [N, 3, H, W] [N, H, W, 3]
    One batch + label (images, labels) (images, labels)

    PyTorch Implementation¶

    üî∏ Folder-based Dataset Loading

    from torchvision import datasets, transforms
from torch.utils.data import DataLoader

# Define transforms
transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406],
                         [0.229, 0.224, 0.225])
])

# Load dataset
train_dataset = datasets.ImageFolder("data/train", transform=transform)
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)

# Check class mapping
print(train_dataset.class_to_idx)  # e.g., {'cat': 0, 'dog': 1}

    üî∏ Iterating Through a Batch

    for images, labels in train_loader:
    print(images.shape)  # [32, 3, 224, 224]
    print(labels)        # Tensor of shape [32]
    break

    TensorFlow Implementation¶

    üî∏ Load from Folder with image_dataset_from_directory

    import tensorflow as tf

train_ds = tf.keras.preprocessing.image_dataset_from_directory(
    "data/train",
    image_size=(224, 224),
    batch_size=32,
    label_mode="int",  # or 'categorical' for one-hot
    shuffle=True
)

# Check output
for images, labels in train_ds.take(1):
    print(images.shape)  # (32, 224, 224, 3)
    print(labels.shape)  # (32,)

    üî∏ Apply Preprocessing via .map()

    def preprocess(img, label):
    img = tf.cast(img, tf.float32) / 255.0
    mean = tf.constant([0.485, 0.456, 0.406])
    std = tf.constant([0.229, 0.224, 0.225])
    img = (img - mean) / std
    return img, label

train_ds = train_ds.map(preprocess)
